# Роадмап — Будущее 2.0

| Этап | Период | Что делаем | Ответственные |
|---|---|---|---|
| 1. Фундамент | 1–3 мес. | Облако+IaC; Data Lake/Lakehouse; Kafka+CDC; Каталог+Линейка; IAM (SSO/MFA, ABAC) | Платформа; Интеграция; Security/IAM; SRE/DevOps |
| 2. Портал (MVP) | 3–6 мес. | Портал доступов; шаблоны пайплайнов; семантика с «золотыми» метриками; DQ в CI/CD; первые Data Products; канарейки | Платформа; BI/Семантика; Governance; Домены; SRE |
| 3. Автономия доменов | 6–9 мес. | EDA/CDC повсеместно; Data Contracts; реестр Data Products; автоскейл; cost‑governance; усиление PHI/PII/PCI | Интеграция; Домены; Governance/Security; Платформа; FinOps |
| 4. Оптимизация/вывод легаси | 9–12 мес. | Перенос критичных отчётов в Lakehouse/семантику; SLO/SLI; оптимизация хранения/compute; план вывода Camel | Платформа/BI; Домены; Governance; FinOps; SRE |


## Обоснование(влияние на бизнес)

| Этап | Зачем бизнесу |
|---|---|
| 1. Фундамент | Быстро запустить платформу и подготовить основу для портала, обеспечить базовую безопасность и управляемость данных |
| 2. Портал (MVP) | Дать бизнесу самообслуживание и единые метрики для ускорения принятия решений |
| 3. Автономия доменов | Развязать домены, сделать интеграции предсказуемыми через контракты и SLA |
| 4. Оптимизация/легаси | Снизить риски и стоимость поддержки, повысить надёжность и стабильность сервисов |
